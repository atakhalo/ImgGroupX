# 开发指南

## 环境要求

- **Node.js** >= 18
- **Rust** >= 1.70
- **VS Code** + Tauri 扩展

## 常用命令

```bash
# 开发模式（热更新）
npm run tauri dev

# 前端 TypeScript 检查
npx vue-tsc --noEmit

# 前端构建
npx vite build

# Rust 编译检查（仅后端）
cd src-tauri && cargo check

# Rust 完整编译
cd src-tauri && cargo build

# 生产构建
npm run tauri build
```

> 注意：`npm run dev` 只启动 Vite 前端，需要用 `npm run tauri dev` 才能启动完整 Tauri 应用。

## 添加新的 Rust Command

1. 在 `src-tauri/src/lib.rs` 中定义函数
2. 添加 `#[tauri::command]` 属性
3. 在 `tauri::generate_handler![]` 中注册
4. 在前端用 `invoke('command_name', { args })` 调用

```rust
#[tauri::command]
fn my_command(input: String) -> Result<String, String> {
    // 实现逻辑
    Ok(result)
}
```

```typescript
// 前端调用
const result = await invoke<string>('my_command', { input: 'value' })
```

## 添加新的工具模块

1. 在 `src/utils/` 下创建 `.ts` 文件
2. 导出函数，在组件中 `import` 使用

```typescript
// src/utils/example.ts
export function doSomething(input: string): string {
  return `hello ${input}`
}
```

## 添加新的 Vue 组件

1. 在 `src/components/` 下创建 `.vue` 文件
2. 使用 `<script setup lang="ts">` + `<template>` + `<style scoped>`
3. TypeScript 类型定义在 `src/types/index.ts`

```vue
<script setup lang="ts">
import { ref } from 'vue'
import type { MyType } from '../types'

const props = defineProps<{
  item: MyType
}>()

const emit = defineEmits<{
  action: [payload: string]
}>()
</script>

<template>
  <div class="my-component">
    {{ item }}
    <button @click="emit('action', 'hello')">Click</button>
  </div>
</template>

<style scoped>
.my-component { /* styles */ }
</style>
```

## 组件通信模式

| 方向 | 方式 |
|------|------|
| 父→子 | Props |
| 子→父 | Emits |
| 跨组件 | `imageStore.ts` 全局 reactive 状态 |

### 事件冒泡模式

递归组件（如 `FolderGroup`）的事件通过逐层 emit 冒泡到 `FolderPanel`，最终到 `App.vue`：

```
子 FolderGroup → emit('toggle')
  → 父 FolderGroup → 转发 emit('toggle', n)
    → FolderPanel → toggleNode()
```

`rootPath` 通过 props 自上而下传递，确保子节点能定位所属根路径。

### 路径排除机制

子节点删除不直接删图片，而是通过事件链通知 App.vue 调用 `excludeSubPath(rootPath, subPath)`：
```
子 FolderGroup → emit('excludeNode', rootPath, subPath)
  → 父 FolderGroup → 转发
    → FolderPanel → 转发
      → App.vue → excludeSubPath() → rootExclusions 更新
        → buildFolderTree 重新构建（跳过排除路径）
```

## 功能开发流程

1. 在 `src/types/index.ts` 中添加类型（如需要）
2. 在 `imageStore.ts` 中添加状态和 API 函数
3. 创建/修改组件实现 UI
4. 在 `App.vue` 中组合组件和事件
5. 必要时添加 Rust Command

## 快捷键添加流程

1. 在 `src/utils/shortcuts.ts` 的 `SHORTCUTS` 数组中添加 `ShortcutDef`（含 id/defaultKey/altKeys/scope/labelKey）
2. 在 `src/i18n/index.ts` 中添加中英文翻译键
3. 在目标组件中引入 `matchShortcut(e, 'actionId')` 替代 `e.key === 'xxx'`
4. 设置页快捷键标签页自动显示，用户可自定义

> 只读操作（如滚轮缩放）设置 `readonly: true`，不会出现在绑定列表中。

## 标记功能开发说明

标记系统包含以下部分：
- **类型**: `ImageItem.markLevel?: MarkLevel`（1-5），`AppSettings.markColors`（5色）
- **Store 函数**: `setImageMark(path, level)`、`setSelectedImagesMark(level)`、`selectImagesByMark(level)`、`deselectImagesByMark(level)`
- **网格显示**: `GridItem.vue` 中 `outline` 显示彩色边框，`mark-badge` 显示角标
- **大图显示**: `ImageViewer.vue` 信息栏显示标记圆标，`OperationBar.vue` 标记菜单
- **控制栏**: `SelectionBar.vue` 标记菜单（标记/全选/取消选择）
- **快捷键**: `shortcuts.ts` 中 `viewer.mark0`~`viewer.mark5`

## 设置项添加流程

1. 在 `src/types/index.ts` 的 `AppSettings` 中添加字段
2. 在 `imageStore.ts` 的 `defaultSettings` 中添加默认值

4. 在 `src/i18n/index.ts` 中添加中英文翻译键

## 文件监听开发说明

### Rust 端
- 使用 `notify` crate 的 `recommended_watcher`（Windows 上为 ReadDirectoryChangesWatcher）
- 监听器实例存储在 `Mutex<Option<RecommendedWatcher>>` 中，通过 `app_handle.state()` 管理
- 单独的 `Mutex<bool>` 控制 suppress 标志，自身操作时抑制事件
- 事件过滤：仅 emit 受支持图片格式的变更路径

### 前端端
- `setupFolderWatcher()` 在扫描文件夹后调用，更新监听范围
- `setSuppressWatcher(true/false)` 在删除操作前后调用
- `fs-changed` 事件携带 `string[]` 路径列表，累积到 `state.pendingChanges`
- `applyFileChanges()` 使用 `normPath()` 规范化路径后比较，兼容 Windows 大小写不敏感

## Tauri v2 注意事项

### 权限配置
在 `src-tauri/capabilities/default.json` 中添加需要的权限：

```json
{
  "permissions": [
    "core:default",
    "dialog:allow-open",
    "opener:default",
    "opener:allow-open-path",
    "opener:allow-reveal-item-in-dir",
    "core:event:default",
    "core:window:default",
    "core:window:allow-close",
    "core:window:allow-set-focus"
  ]
}
```

### 拖拽
在 `tauri.conf.json` 窗口配置中确保 `"dragDropEnabled": true`。
使用 `getCurrentWindow().onDragDropEvent()` 监听拖入事件：

```typescript
import { getCurrentWindow } from '@tauri-apps/api/window'

const unlisten = await getCurrentWindow().onDragDropEvent((event) => {
  if (event.payload.type === 'drop') {
    // event.payload.paths 包含拖入的文件/文件夹路径
  }
})
```

### 对话框
使用 `@tauri-apps/plugin-dialog` 的 `open()` 代替 HTML 文件输入：

```typescript
import { open } from '@tauri-apps/plugin-dialog'

// 选择文件夹
const folders = await open({ directory: true, multiple: true })

// 选择文件
const files = await open({ 
  multiple: true, 
  filters: [{ name: '图片', extensions: ['jpg', 'png'] }] 
})
```

### 文件打开
```typescript
import { revealItemInDir, openPath } from '@tauri-apps/plugin-opener'

await revealItemInDir(path)  // 资源管理器定位
await openPath(path)         // 默认程序打开
```

### 外部程序打开
```typescript
// Rust 端
#[tauri::command]
fn open_with_program(path: String, program: String) -> Result<(), String> { ... }

// 前端
await invoke('open_with_program', { path, program })
```

### 国际化

翻译字典位于 `src/i18n/index.ts`，新增文本需同时添加中英文条目。
所有用户可见文本通过 `t(key)` 翻译。新增文本步骤：
1. 在 `src/i18n/index.ts` 的 `zh` 和 `en` 字典中添加翻译
2. 模板中使用 `{{ $t('key') }}`，脚本中使用 `import { t } from '../i18n'; t('key')`
3. 需要参数时使用 `$t('key', { n: 5 })`，替换 `{n}` 占位符

### 命令行参数

启动时自动读取 `std::env::args()`（跳过 exe 路径），传入 `handleDroppedPaths` 处理，支持文件夹和图片文件。

### 配置文件
设置自动保存到 exe 同目录的 `config-user.yaml`。通过 `serde_yaml` 序列化，`load_config` / `save_config` 两个 command 读写。
