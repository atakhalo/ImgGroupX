# 技术架构

## Rust 后端 (Tauri Commands)

### 命令列表

| 命令 | 功能 | 参数 | 返回 |
|------|------|------|------|
| `scan_folder` | 扫描单文件夹(递归) | `path: String` | `ScanResult { images, total }` |
| `scan_folders` | 批量扫描 | `paths: Vec<String>` | `Vec<ScanResult>` |
| `load_image_base64` | 读取图片为 base64 | `path: String` | `String (data:image/...;base64,...)` |
| `get_image_metadata` | 读取 EXIF 元数据(含 UserComment 解码) | `path: String` | `Vec<Vec<String>>` |
| `open_in_explorer` | 系统默认方式打开文件 | `path: String` | `()` |
| `open_with_program` | 用指定程序打开文件 | `path: String, program: String` | `()` |
| `check_is_dir` | 判断路径是否为目录 | `path: String` | `bool` |
| `get_cli_args` | 获取命令行参数 | - | `Vec<String>` |
| `load_config` | 读取 config-user.yaml | - | `serde_json::Value` |
| `save_config` | 保存 config-user.yaml | `settings: Value` | `()` |

### 依赖 Crate

| Crate | 用途 |
|-------|------|
| `walkdir` | 递归遍历文件夹 |
| `base64` | 图片数据编码 |
| `imagesize` | 快速获取图片尺寸（无需解码全图） |
| `kamadak-exif` | 读取 EXIF 元信息 |
| `chrono` | 时间格式化 |
| `serde` / `serde_json` | 序列化通信 |
| `serde_yaml` | 配置文件 YAML 序列化 |
| `tauri-plugin-dialog` | 原生文件/文件夹对话框 |
| `tauri-plugin-opener` | 系统打开/资源管理器定位 |

### EXIF UserComment 解码

参照 piexif 实现：跳过 8 字节编码前缀（ASCII/UNICODE/JIS），尝试 UTF-8 解码，失败降级 Latin-1。`Undefined` 类型值统一走此解码逻辑。

### 配置持久化

- 配置文件路径：`exe 所在目录/config-user.yaml`
- 格式：YAML
- 启动时自动加载，设置变更后 500ms 防抖自动保存

### 图片处理流程

```
文件夹选择 → Rust scan_folder
  → WalkDir 递归遍历
  → 过滤支持的图片格式
  → 读取尺寸(imagesize)、修改时间
  → 返回 Vec<ImageInfo>
  → 前端按需调用 load_image_base64
  → Rust 读取文件 → base64 编码 → 返回 data URI
```

## 前端架构

### 国际化 (i18n)

- 模块：`src/i18n/index.ts`，包含中英双语翻译字典
- `t(key, params?)` 函数读取 `state.settings.language` 决定当前语言
- 通过 `main.ts` 注册为 Vue 全局属性 `$t`，所有组件模板中直接使用
- 响应式：`t()` 内部读取 reactive 状态，语言切换后所有使用 `computed` 或模板表达式的文本自动更新
- 类型声明：`src/i18n-types.d.ts`

### 状态管理

使用 Vue 3 `reactive()` 全局状态对象 (`imageStore.ts`)，包含：

```
state
├── allImages          # 所有图片列表（去重）
├── loadedRootPaths    # 用户加载的根文件夹路径
├── folderTree         # 目录树结构
├── virtualGroups      # 虚拟分组列表
├── folderGroup        # 是否启用目录分组
├── showGroupTitle     # 是否显示分组标题（含虚拟分组）
├── selectMode         # 选择/查看模式
├── selectedPaths      # 选中图片路径 Set
├── settings           # 应用设置（含颜色、外部程序等）
└── loading            # 加载状态

rootExclusions (reactive Map)  # 根路径 → 排除子路径集合
```

### 数据流向

```
Rust 扫描 → imageStore.scanFolders()
  → state.allImages 更新
  → FolderPanel 通过 computed 调用 buildFolderTree()
    → 根据 loadedRootPaths + rootExclusions 构建多根树
  → GridItem 按需调用 loadImageBase64()
    → Rust 返回 base64 → 显示图片
```

Shift+Click 滚轮 → App.vue content-area → state.settings.gridSize 调整 → 全局格子大小变更

### 树构建算法

`buildFolderTree(images, rootPaths)`:

1. 按 `loadedRootPaths` 分组，每个根文件夹独立建树
2. 检查 `rootExclusions` 排除被手动删除的子路径
3. 根节点显示完整路径，子节点仅显示文件夹名
4. 虚拟分组作为根节点显示在树顶（📦 标记）
5. 没有根路径时（仅加载单图）返回空树

### 子节点删除机制（路径排除）

删除子节点不直接删除图片，而是将子路径加入 `rootExclusions`（该根路径的排除集）。树构建时自动跳过被排除路径的图片。

- 根节点删除：从 `loadedRootPaths` 移除，清理对应 exclusions。只删除不被其他根路径覆盖的图片
- 子节点删除：将相对路径加入父根的 exclusions。图片保留在 `allImages` 中，仍可被其他根路径引用

### 展开状态管理

使用 `reactive(Map<string, boolean>)` 管理节点展开状态：
- 键为节点路径，值为是否展开
- 所有节点默认展开
- 筛选变更时自动更新：有匹配 → 展开，无匹配 → 折叠
- 筛选清空时全部恢复展开

### 对比视图

选中 2 张图片后点击「对比」按钮进入。两图重叠显示，左图在上通过 `clip-path: inset()` 裁剪，可拖拽分割线调整裁剪比例。底部悬浮操作栏支持缩放、平移、适应窗口。Ctrl+滚轮同步缩放两图。

### 大图分组内导航

- GridView 点击时通过 `viewImage` 事件携带 scope（当前网格图片列表）
- 分组模式下使用 scope 作为导航范围，图片信息 (i/n) 和左右切换限于小分组内
- 未分组模式下使用全部图片导航
- Scope 使用排序+筛选后的 `processedImages`，确保与网格显示顺序一致
