### EXIF UserComment 解码

参照 piexif 实现：跳过 8 字节编码前缀（ASCII/UNICODE/JIS），尝试 UTF-8 解码，失败降级 Latin-1。`Undefined` 类型值统一走此解码逻辑。

### 图片处理流程

#### 全量扫描（refreshFolders / 旧流程）

```
文件夹选择 → Rust scan_folders
  → WalkDir 递归遍历
  → 过滤支持的图片格式
  → 读取尺寸(imagesize)、修改时间
  → 返回 Vec<ScanResult>
  → 前端一次加入 state.allImages
  → buildFolderTree 构建树
  → GridItem 按需调用 load_image_base64
```

#### 图片按需加载（loadImageBase64）

```
GridItem IntersectionObserver 进入视口（提前 300px）
  → loadImageBase64(item, force=false)
    ├─ 有缓存 base64 → 直接返回
    ├─ 大小阈值检查（非 force 时）
    │   ├─ 超过 maxLoadSizeMB → 设置 item.error 抛错，显示跳过占位符
    │   └─ 未超过 → 继续
    ├─ 设置 item.loading = true
    ├─ invoke('load_image_base64') → Rust 读取+编码
    ├─ 成功 → 设置 item.base64，显示图片
    └─ 失败 → 设置 item.error，显示错误/跳过占位符

大图查看器 / 对比视图
  → loadImageBase64(item, loadSkippedOnView)
    ├─ loadSkippedOnView=true → force=true，跳过大小检查直接加载
    └─ loadSkippedOnView=false → 不尝试加载，显示跳过占位符
```

#### 渐进式扫描（默认流程）

```
文件夹选择 → Rust scan_folders_progressive（后台线程）
  → WalkDir 遍历，每扫完一个目录立即通过事件发送
  → 前端监听 "scan-dir-progress" 事件 → handleDirProgress()
  → 逐批加入 state.allImages → 树实时更新
  → 网格逐步渲染，图片按需加载
  → 全部完成后 emit "scan-all-complete" → handleScanComplete()
  → 自动启动文件变更监听 setupFolderWatcher()
```

### 文件变更监听流程

```
用户添加文件夹 → setupFolderWatcher()
  → Rust update_watcher (notify::recommended_watcher)
  → 递归监听所有根路径
  → 文件变更时 → 过滤支持格式 → emit "fs-changed" 事件（携带路径列表）
  → 前端累积到 pendingChanges
  → 用户点击刷新
    ├─ 有红点 → applyFileChanges(pendingChanges) 细粒度增/删/改
    └─ 无红点 → refreshFolders() 全量重扫
```

### 剪贴板图片复制

使用 Web Clipboard API (`navigator.clipboard.write`) 实现图片复制到剪贴板：

```
data URI → 解 base64 → Blob → 检查 MIME
  ├─ PNG: 直接 new ClipboardItem({ 'image/png': blob })
  └─ 其他格式 (webp/jpeg/...): createImageBitmap(blob) → canvas → canvas.toBlob('image/png') → ClipboardItem
```

- SVG 格式特殊处理：提取 `<svg>` 标签内容构造 `image/svg+xml` Blob
- 无需任何 Rust 依赖或插件

### 数据流向

```
Rust 扫描 → imageStore.scanFolders()
  → state.allImages 更新
  → setupFolderWatcher() → Rust update_watcher → 开始监听文件变更
  → FolderPanel 通过 computed 调用 buildFolderTree()
    → 根据 loadedRootPaths + rootExclusions 构建多根树
  → GridItem 按需调用 loadImageBase64()
    → Rust 返回 base64 → 显示图片
```

Ctrl+滚轮 → App.vue content-area → state.settings.gridSize 调整 → 全局格子大小变更

文件外部变更 → notify 监听 → emit "fs-changed" 事件（携带路径列表）→ state.pendingChanges 累积
→ 用户点击刷新
  ├─ 有红点 → applyFileChanges() 细粒度增/删/改
  └─ 无红点 → refreshFolders() 全量重扫

### 树构建算法

`buildFolderTree(images, rootPaths)`:

1. 按 `loadedRootPaths` 分组，每个根文件夹独立建树
2. 检查 `rootExclusions` 排除被手动删除的子路径
3. 根节点显示完整路径，子节点仅显示文件夹名
4. 虚拟分组作为根节点显示在树顶（📦 标记）
5. 没有根路径时（仅加载单图）返回空树

### 子节点删除机制（路径排除）

删除子节点不直接删除图片，而是将子路径加入 `rootExclusions`（该根路径的排除集）。树构建时自动跳过被排除路径的图片。

- 根节点删除：从 `loadedRootPaths` 移除，清理对应 exclusions。只删除不被其他根路径覆盖的图片
- 子节点删除：将相对路径加入父根的 exclusions。图片保留在 `allImages` 中，仍可被其他根路径引用

### 着色体系

FolderGroup 采用四层着色体系（通用逻辑，不限于彩虹模式）：

| 层级 | 元素 | 不透明度 |
|------|------|---------|
| 整体背景 | `folder-group` | 100%（直接使用 base color） |
| 标题条 | `folder-header` | transparent（依赖父级） |
| 网格包裹 | `folder-grid-wrapper` | transparent |
| 网格容器 | `grid-container`（通过 prop） | 彩虹时 100%，否则 `bgColor` |

- `getBaseColor(depth)` 统一获取基础色：彩虹模式取彩虹轮替 `colors[(depth-1) % N]`，否则取 `childTitleBgColor`
- depth=0（根节点）保持 `rootTitleBgColor`
- 颜色不足时自动轮替：1,2,3,4,5,1,2,...

### 展开状态管理

**架构**：`expandedMap`（`reactive(Map)`）存储每个节点的展开状态，首次构造时确定，用户可翻转，永不重算。

**大文件夹自动折叠**：
- 每个根路径有独立的 `rootLargeDetected` 标志
- 节点首次构造时，通过树搜索找到所属根节点
- 若该根已标记 → 节点默认折叠
- 否则展开，然后检查根节点是否达标（>5 子文件夹 或 >100 张图）
- 达标则标记该根，当前节点也折叠
- 此后该根下所有新节点默认折叠
- `state.loading` 从 true→false 时重置所有标志

**筛选联动**：筛选变更时自动更新——有匹配的节点展开，无匹配的折叠。

### 对比视图

选中 2 张图片后点击「对比」按钮进入。两图重叠显示，左图在上通过 `clip-path: inset()` 裁剪，可拖拽分割线调整裁剪比例。底部悬浮操作栏支持缩放、平移、适应窗口。Ctrl+滚轮同步缩放两图。

### 大图分组内导航

- GridView 点击时通过 `viewImage` 事件携带 scope（当前网格图片列表）
- 分组模式下使用 scope 作为导航范围，图片信息 (i/n) 和左右切换限于小分组内
- 未分组模式下使用全部图片导航
- Scope 使用排序+筛选后的 `processedImages`，确保与网格显示顺序一致
